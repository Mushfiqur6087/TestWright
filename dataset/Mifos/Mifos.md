## Navigation

Mifos is a comprehensive microfinance and core banking platform built on Apache Fineract. The application uses a Material Design layout with a left icon-based sidebar and a main content area. After login, a persistent top navigation bar displays the Mifos logo on the left and menu items: Institution, Accounting, Reports, Admin, Self Service, and Configuration Wizard, along with icons for global search, language selection, notifications, and user profile on the right. The Institution menu contains links to Clients, Groups, Centers, Accounting, Reports (with sub-items: All, Clients, Loans, Savings, Funds, Accounting, XBRL), Admin, and Self Service. The left sidebar contains icon-based shortcuts to commonly used sections of the system (including Clients, Groups, Centers, Products, and more). Each icon navigates to the corresponding page in the main content area. The Products hub (at /products) is a landing page listing all product categories: Loan Products, Savings Products, Share Products, Charges, Collateral Management, Delinquency Buckets, Products Mix, Fixed Deposit Products, Recurring Deposit Products, Manage Tax Configurations, and Floating Rates. Breadcrumbs appear at the top of the content area showing the navigation path. Entity listing pages display data in Material Design tables with search, filter, sort, and pagination controls. Detail pages display entity information in card layouts with action buttons for state transitions. Forms use Material Design input fields with inline validation.

## Functional Overview

### 1. Login

The Login page has two panels. The left panel displays a background image with the title "Mifos X" and descriptive text. The right panel contains the authentication interface. The login panel shows the Mifos logo at the top and a language selector in the top-right corner. The form includes a Tenant dropdown (default value "default"), Username text input, Password masked input, and a "Remember me" checkbox. The "Login" button submits the credentials and becomes enabled only after the required fields are filled. A "Forgot Password?" link appears below the button. If valid credentials are entered, the user is authenticated and redirected to the Dashboard. If credentials are invalid, an error message is displayed. If required fields are empty, validation messages appear next to the fields. Footer links "Resources", "Community", and "Contribute" are displayed at the bottom along with system version information.

### 2. Home Page

The Home page appears after successful login and serves as the main entry point to the system. A top navigation bar provides access to modules including Institution, Accounting, Reports, Admin, Self Service, and Configuration Wizard, along with icons for global search, language selection, notifications, and the user profile menu. A vertical sidebar on the left provides quick navigation through icon-based shortcuts to commonly used sections of the system. The main content area displays a welcome card with the message "Welcome, mifos!" and a "Search Activity" input field that allows users to search for recent system activities. A "Dashboard" button provides quick access to the dashboard view. System version information for Mifos and Fineract is displayed at the bottom of the page.

### 3. Dashboard

The Dashboard page is accessed from the "Dashboard" button on the Home Page. It displays system overview information in a card-based layout. A search field labeled "Search Activity" appears at the top of the page. The main section shows a "Client Trends" chart that visualizes client growth over time for the selected office (e.g., Head Office). The chart includes legends for "New Clients" and "Closed Clients". Below the chart, two summary cards display financial metrics: "Amount Pending / Disbursed" and "Amount Collected". Each card shows data for the selected office and displays "No Data" if no information is available. The page also includes breadcrumb navigation at the top (Dashboard > Home > Dashboard). The top navigation bar and left sidebar remain visible, allowing access to other system modules.

### 4. Global Search

Global search is available from the search icon in the top toolbar and can be accessed from any page after login. Clicking the search icon opens a search input field. Users can enter keywords such as a client name, account number, or ID. As the user types, the system searches across entities including Clients, Groups, Loans, and Savings accounts. Matching results appear in a dropdown list grouped by entity type. Each result shows the entity name, identifier, and status. Selecting a result navigates directly to the corresponding detail page. If no matches are found, a "No results found" message is displayed. The search supports partial matching and is case-insensitive.

### 5. Client Management

The Clients page is accessed from the left sidebar. The breadcrumb shows Home / Clients. The page displays a data table listing all clients with columns for Name (clickable link), Account No., External ID, Status (Pending as yellow chip, Active as green chip, Closed as gray chip, Rejected as red chip, Withdrawn as orange chip), Office, and Staff. A search field allows searching by name, account number, external ID, or mobile number. A filter bar supports filtering by status. If no clients exist, the message "No client was found" is displayed. Pagination controls show total count and page navigation. Buttons at the top-right provide "Import Client" and "Create Client" actions. The Bulk Import page allows downloading a client Excel template, uploading it, and displays an import history table with columns for Name, Import Time, End Time, Completed, Total Records, Success Count, Failure Count, and Download. The Create Client form (breadcrumb: Home / Clients / Create) is a multi-step wizard with Previous and Next navigation. Step 1 — General Details: Office (dropdown, required — shows "Office is required"), Legal Form (dropdown), First Name (text input, required — shows "First name is required"), Middle Name (text input), Last Name (text input, required — shows "Last name is required"), Date of Birth (date picker), Gender (dropdown), Staff (dropdown), Mobile Number, Email Address, Client Type, Client Classification, External ID (must be unique — duplicates produce a validation error), Submitted On (date picker, required), Is Staff (checkbox), Active (checkbox), and Open Savings Account option. Step 2 — Address Details. Step 3 — Family Members (relationship and details). Step 4 — Identifiers (Document Type dropdown and Document Key). "Submit" creates the client in "Pending" status. The Client Detail page displays a header with client name, account number, status badge (colored), activation date, and office. Action buttons change based on status: for Pending — Activate (Activation Date required, must not be before submission date), Edit, Reject (with reason), Withdraw (with reason); for Active — Edit, Transfer Client (destination office — same office is blocked), Close (closure reason required — cannot close with active accounts), Add Charge, New Loan, New Savings, New Share Account; for Closed — Reactivate; for Rejected/Withdrawn — no actions. Tabs show: General, Accounts (sub-tabs for Loans, Savings, Shares, Fixed Deposits, Recurring Deposits), Identifiers (Document Type and Key with add/remove — duplicates prevented), Family Members (with Add action), Notes (chronological with Add Note), and Documents (upload with description).

### 6. Group Management

The Groups page is accessed from the left sidebar. The breadcrumb shows Home / Groups. The page displays a data table listing all groups with columns for Group Name (clickable link), Account Number, Office, Status (Active, Pending, Closed as colored chips), and Staff. Buttons at the top-right allow importing groups or creating a new group. The Create Group form (breadcrumb: Home / Groups / Create) contains fields for Name (text input, required), Office (dropdown, required), Staff (dropdown), Submitted On (date picker, required), Active (checkbox), External Id (text input), and Add Clients (search-and-add interface to associate existing clients). Required fields show validation errors if empty. The Bulk Import Groups page is at Home / Organization / Bulk Import / Groups and has two panels: Groups Template (Office and Staff dropdowns with a Download button to get the Excel template) and Groups upload (Browse file picker and Upload button). A Documents table below shows import history with columns for Name, Import Time, End Time, Completed, Total Records, Success Count, Failure Count, and Download. The Group Detail page displays the group name, account number, status, office, and staff at the top with action buttons (Activate, Edit, Close, Assign Staff, Transfer Clients). Below, tabs show Members (list of client members with links to their profiles), Loan Accounts (group loans and GLIM — Group Linked Individual Micro-loans), Savings Accounts (group savings and GSIM accounts), Notes, and a Calendar/Meeting section for scheduling and recording group meetings. The Collection Sheet feature accessible from a group allows generating a sheet showing all group clients with loan repayment amounts due and savings deposit amounts for batch data entry.

### 7. Center Management

The Centers page is accessed from the left sidebar. The breadcrumb shows Home / Centers. Centers are higher-level organizational units that contain groups. The page displays a data table listing all centers with columns for Name (clickable link), Account #, External Id, Status (colored indicator), and Office Name. Buttons at the top-right provide Import Center and Create Center actions. The Create Center form (breadcrumb: Home / Centers / Create) contains fields for Name (text input, required — shows "Center Name is required" validation), Office (dropdown, required), Staff (dropdown), Active (checkbox), External Id (text input), Submitted On (date picker, required), and Select and Add groups (dropdown search to associate existing groups). Required fields show inline validation errors. The Bulk Import Centers page is at Home / Organization / Bulk Import / Centers and has two panels: Centers Template (Office and Staff dropdowns with a Download button) and Centers upload (Browse file picker and Upload button). A Documents table shows import history with columns for Name, Import Time, End Time, Completed, Total Records, Success Count, Failure Count, and Download. The Center Detail page displays the center name, status, office, and staff with action buttons (Activate, Edit, Close, Assign Staff). Tabs below show Groups (list of member groups with links), Loan Accounts, Savings Accounts, Notes, and a Calendar/Meeting section. The Collection Sheet feature generates a sheet showing all groups and their clients with loan repayment amounts due and savings deposit amounts for batch data entry.

### 8. Loan Products

The Loan Products page is accessed from the left sidebar under Products. The breadcrumb shows Home / Products / Loan Products. The page displays a Filter bar and a data table listing all configured loan products with columns for Name (clickable link), Short Name, Expiry Date, and Status. A "+ Create Loan Product" button at the top-right opens the Create Loan Product form, which is a 6-step stepper wizard with steps: DETAILS, CURRENCY, SETTINGS, TERMS, CHARGES, and ACCOUNTING. Navigation between steps uses Previous and Next buttons at the bottom. The DETAILS step includes Product Name (text input, required), Short Name (text input, required), External Id (text input, optional), Fund (dropdown), Include in Customer Loan Counter (checkbox), Start Date (date picker), Close Date (date picker), and Description (textarea). The CURRENCY step includes currency selection with decimal places and multiples of rounding, and principal amount with minimum, default, and maximum numeric inputs. The SETTINGS step includes Amortization method (dropdown — Equal Installments, Equal Principal Payments), Interest Method (dropdown — Flat, Declining Balance), Interest Calculation Period (dropdown — Daily, Same as repayment period), Repayment Strategy (dropdown), Days in Year (dropdown — 360, 365, Actual), Days in Month (dropdown — 30, Actual), and numeric fields for Grace on Principal Payment, Grace on Interest Payment, Grace on Interest Charged, Interest Free Period, and Arrears Tolerance. The TERMS step includes Number of Repayments with minimum, default, and maximum, Repaid Every frequency number with unit dropdown (Days, Weeks, Months), and Nominal Interest Rate with minimum, default, and maximum fields. The CHARGES step allows adding predefined charges/fees with a search-and-add interface. The ACCOUNTING step provides radio button selection for None, Cash-based, Accrual (periodic), or Accrual (upfront), and when selected shows dropdown mappings for Fund Source, Loan Portfolio, Interest on Loans, Income from Fees, Income from Penalties, Losses Written Off, and Overpayment Liability GL accounts. All required fields show inline validation errors. Clicking an existing product name opens its detail view with an Edit option.

### 9. Savings Products

The Saving Products page is accessed from the left sidebar under Products. The breadcrumb shows Home / Products / Saving Products. The page displays a Filter bar and a data table listing all savings products with columns for Name (clickable link) and Short Name. A "+ Create Savings Product" button at the top-right opens the Create Savings Product form. A create button opens the Create Savings Product form, which is a stepper wizard with steps: DETAILS, CURRENCY, SETTINGS, TERMS, CHARGES, and ACCOUNTING. Navigation between steps uses Previous and Next buttons. The DETAILS step includes Product Name (text input, required), Short Name (text input, required), Description (textarea), and External Id. The CURRENCY step includes Currency (dropdown), Decimal Places (numeric), and Currency In Multiples Of (numeric). The TERMS step includes Nominal Annual Interest Rate (numeric input), Interest Compounding Period (dropdown — Daily, Monthly, Quarterly, Semi-Annual, Annually), Interest Posting Period (dropdown — Monthly, Quarterly, Biannually, Annually), Interest Calculated Using (dropdown — Daily Balance, Average Daily Balance), and Days in Year (dropdown — 360, 365, Actual). The SETTINGS step includes Minimum Opening Balance (numeric input), Lock-in Period Frequency (numeric input and dropdown), Apply Withdrawal Fee for Transfers (checkbox), Minimum Balance Required for Interest Calculation (numeric), Enforce Minimum Required Balance checkbox with Minimum Required Balance field, Is Overdraft Allowed checkbox with Maximum Overdraft Amount and Nominal Annual Interest Rate for Overdraft fields, Enable Withhold Tax checkbox with Tax Group dropdown, and Enable Dormancy Tracking checkbox with Days to Inactive, Days to Dormancy, and Days to Escheat fields. The CHARGES step allows adding predefined charges with a search-and-add interface. The ACCOUNTING step provides radio button selection for None or Cash-based, and when Cash-based is selected shows GL account dropdown mappings for Savings Reference, Savings Control, Transfers in Suspense, Interest on Savings, Income from Fees, Income from Penalties, and Escheat Liability. All required fields show inline validation errors. Clicking an existing product name opens its detail view with an Edit option. Fixed Deposit Products and Recurring Deposit Products are also accessible from the left sidebar under Products as separate pages, each with a similar stepper wizard. Fixed Deposit Products include additional steps for Pre-Closure (Pre-Mature Closure Applicable checkbox, Pre-Closure Penal Interest numeric input, Pre-Closure Penal Interest On dropdown) and Deposit Term (Minimum Deposit Term with unit, Maximum Deposit Term with unit, In Multiples Of Deposit Term, Minimum Deposit Amount, Maximum Deposit Amount, Default Deposit Amount) and an Interest Rate Chart step (tiered rates each with Period From, Period To, Period Type, Interest Rate, and optional Amount Range slabs). Recurring Deposit Products additionally include Mandatory Recommended Deposit Amount (numeric input), Is Mandatory Deposit checkbox, Allow Withdrawal checkbox, Adjust Advance Towards Future Payments checkbox, and Recurring Frequency with unit dropdown.

### 10. Share Products

The Share Products page is accessed from the left sidebar under Products. The breadcrumb shows Home / Products / Share Products. The page displays a data table listing all configured share products with columns for Product Name (clickable link), Short Name, and Total Shares. A create button opens the Create Share Product form, which is a 7-step stepper wizard with steps: DETAILS, CURRENCY, TERMS, SETTINGS, MARKET PRICE, CHARGES, and ACCOUNTING. Navigation between steps uses Previous and Next buttons. The DETAILS step includes Product Name (text input, required), Short Name (text input, required), and Description (textarea, required). The CURRENCY step includes Currency (dropdown), Decimal Places, and Currency In Multiples Of. The TERMS step includes Total Number of Shares (numeric input, required), Shares to be Issued (numeric input), Nominal/Unit Price (numeric input, required), and Capital Value (auto-calculated). The SETTINGS step includes Allow Dividends for Inactive Clients (checkbox), Minimum Shares per Client (numeric input), Maximum Shares per Client (numeric input), Nominal Shares per Client (numeric input), Minimum Active Period Frequency (numeric input and dropdown), and Lock-in Period (numeric input and dropdown). The MARKET PRICE step provides a table to define market price periods — each row has a From Date (date picker) and Share Value (numeric input); rows can be added or removed. The CHARGES step allows adding predefined charges with a search-and-add interface. The ACCOUNTING step provides radio button selection for None or Cash-based, and when Cash-based is selected shows GL account dropdown mappings for Share Reference, Share Suspense, Equity in Shares, Income from Fees, and Share Equity. All required fields show inline validation errors. Clicking an existing product opens its detail view with Edit and Delete options.

### 11. Charges

The Charges page is accessed from the left sidebar under Products. The breadcrumb shows Home / Products / Charges. The page displays a data table of all charge definitions with columns for Name (clickable link), Charge Applies To, Is Penalty, Is Active, and Is Paid Derived. A "+ Create Charge" button at the top-right opens the Create Charge form with fields for Charge Name (text input, required), Charge Applies To (dropdown — Loan, Savings Account, Client, Shares, required), Currency (dropdown, required), Charge Time Type (dropdown — varies by entity: for Loans includes Disbursement, Specified Due Date, Installment Fee, Overdue Fees, Tranche Disbursement; for Savings includes Specified Due Date, Savings Activation, Withdrawal Fee, Annual Fee, Monthly Fee, Overdraft Fee), Charge Calculation Type (dropdown — Flat, Percentage of Amount, Percentage of Interest, Percentage of Total Outstanding), Amount (numeric input, required), Is Penalty (checkbox), Is Active (checkbox), Tax Group (dropdown, optional), and Payment Mode (dropdown — Regular, Account Transfer). All required fields show inline validation errors if empty. "Submit" creates the charge definition which can then be associated with loan or savings products. Clicking an existing charge name opens its detail view with Edit and Delete options.

### 12. Floating Rates

The Floating Rates page is accessed from the left sidebar under Products. The breadcrumb shows Home / Products / Floating Rates. The page displays a data table listing all configured floating rate definitions with columns for Floating Rate Name (clickable link), Is Base Lending Rate indicator, Is Active indicator, and Created By. A "+ Create Floating Rate" button opens the Create Floating Rate form with fields for Floating Rate Name (text input, required), Is Base Lending Rate (checkbox — only one base rate can exist in the system), and Is Active (checkbox). The Rate Periods section allows defining time-segmented interest rates — each row has From Date (date picker), Interest Rate (numeric input, percentage), and Is Differential Rate checkbox (if checked, the rate is added to or subtracted from the base lending rate rather than used as an absolute value). Multiple rate periods can be added to define rate changes over time. "Submit" creates the floating rate definition. Floating rates are referenced by Loan Products when configuring variable interest rates — when a loan product is linked to a floating rate, the loan's interest rate adjusts automatically based on the applicable rate period. The detail view shows the full rate history with Edit option to add new periods or modify existing ones.

### 13. Delinquency Management

The Delinquency pages are accessed from the left sidebar under Products. The breadcrumb shows Home / Products / Delinquency Buckets. Two sub-pages exist: Delinquency Ranges and Delinquency Buckets. The Delinquency Ranges page displays a data table with columns for Classification (clickable link), Minimum Age Days, and Maximum Age Days. A "+ Create" button opens the Create Delinquency Range form with fields for Classification (text input, required), Minimum Age Days (numeric input, required), and Maximum Age Days (numeric input, optional — if blank, applies to all days beyond minimum). The Delinquency Buckets page displays a data table with columns for Bucket Name (clickable link) and associated ranges. A "+ Create" button opens the Create Delinquency Bucket form with Bucket Name (text input, required) and allows adding multiple delinquency ranges in sequence (e.g., 1–29 days = Range 1, 30–59 days = Range 2, 60+ days = Range 3). Delinquency buckets are linked to Loan Products in the product configuration — when a loan becomes overdue, the system automatically classifies it into the appropriate delinquency range based on the number of days past due. This classification is used for portfolio at risk reporting, provisioning calculations, and collection workflow triggers. Clicking an existing bucket or range opens its detail view with Edit and Delete options.

### 14. Loan Account

A new loan application is initiated from the Client Detail page using action buttons in the client header area. The Loan Application form is a multi-step wizard. Step 1 — Details: Product Name (dropdown listing all active loan products — selecting a product auto-populates default values), Loan Officer (dropdown), Loan Purpose (dropdown from configured code values), Fund (dropdown), Submitted On date (date picker), Expected Disbursement Date (date picker), Principal (numeric input, bounded by product min/max), Number of Repayments (numeric input), Repaid Every (frequency and unit), Interest Rate (numeric input, bounded by product min/max), and External ID. Step 2 — Terms: Repayment strategy, Amortization, Interest Method, and Interest Calculation Period dropdowns pre-filled from the product but adjustable, plus various grace period fields. Step 3 — Charges: A table listing charges inherited from the product with amount, charge time, and calculation type, plus an "Add Charge" button for additional charges. Step 4 — Collateral (optional): Add collateral items with Collateral Type, Value, and Description. "Submit" creates the loan in "Submitted and Pending Approval" status. The Loan Detail page header displays the loan account number, product name, client name (clickable link), loan status badge (Submitted and Pending Approval in yellow, Approved in blue, Active in green, Closed in gray, Overpaid in purple, Written Off in red), and loan balance. An action button bar provides lifecycle transition buttons based on status: for Pending Approval — Approve (opens dialog with Approved On Date, Approved Amount, Expected Disbursement Date, and Note fields), Reject, Withdraw, Delete; for Approved — Disburse (opens form with Disbursed On Date, Transaction Amount, Payment Type dropdown listing Cash/Check/Mobile Money/Bank Transfer, Account Number, Check Number, Receipt Number, and Bank Number fields; a "Disburse to Savings" option deposits directly into the client's linked savings), Undo Approval; for Active — Make Repayment (opens form with Transaction Date, Transaction Amount pre-filled with amount due, Payment Type dropdown, and payment detail fields; partial repayments are applied per the configured repayment strategy; full settlement changes status to "Closed obligations met"), Waive Interest, Write Off (records the total outstanding as a loss and closes the loan), Close, Reschedule (opens request form with Reschedule From Date, Reason, Adjusted Due Date, Grace on Principal/Interest, Extra Terms, and New Interest Rate — creates a pending request requiring separate approval), Prepay Loan, Foreclosure, Charge Off, Assign Loan Officer. The main content tabs include: Summary (disbursement details, repayment schedule summary, total paid, total outstanding, total overdue, charges summary), Repayment Schedule (table with columns for Installment Number, Due Date, Principal Due, Interest Due, Fees Due, Penalties Due, Total Due, amounts paid for each, Total Outstanding, and status indicators), Transactions (table of all loan transactions with Date, Type, Amount, Principal/Interest/Fees/Penalties portions, Outstanding Balance, and details/undo link), Charges (applied charges with charge name, amount, due date, paid, waived, outstanding), Collateral, Notes, and Documents.

### 15. Savings Account

A new savings account is created from the Client Detail page using action buttons in the client header area. The form includes Product Name (dropdown listing all active savings products — selecting a product auto-populates default values), Field Officer (dropdown), Submitted On date (date picker), Nominal Annual Interest Rate (numeric input, pre-filled from product), Interest Compounding Period, Interest Posting Period, Interest Calculated Using, Days in Year dropdowns (all pre-filled), Minimum Opening Balance, Lock-in Period, and Allow Overdraft checkbox. A Charges section lists product charges and allows adding additional charges. "Submit" creates the account in "Submitted and Pending Approval" status. The Savings Account Detail page displays the account number, product name, client name, status badge (Submitted and Pending Approval, Approved, Active, Dormant, Closed, Blocked), account balance, and available balance. Action buttons change based on status: for Pending — Approve, Reject, Withdraw Application; for Approved — Activate, Undo Approval; for Active — Deposit (opens form with Transaction Date, Transaction Amount, Payment Type dropdown with Cash/Check/Mobile Money options, Account Number, Check Number, Routing Code, Receipt Number, Bank Number, and Note fields; submitting credits the savings account), Withdraw (same form as Deposit; system validates withdrawal does not exceed available balance unless overdraft is enabled; if minimum balance is enforced and withdrawal would breach it, an error is shown), Post Interest, Calculate Interest, Close, Block Account, Block Debit, Block Credit. Tabs show: Summary (account details and interest summary), Transactions (table of all deposits, withdrawals, and interest postings with date, type, amount, and running balance), Charges (applied charges with status), and Documents.

### 16. Share Account

A new share account is created from the Client Detail page using action buttons in the client header area. The Share Account Application form includes Share Product (dropdown listing all active share products — selecting a product auto-populates default values), Submitted On date (date picker), Requested Shares (numeric input — number of shares the client wishes to purchase, bounded by product min/max per client), Application Date (date picker), Savings Account for Charges (dropdown listing the client's active savings accounts to debit fees from), and External ID. A Charges section lists product-inherited charges and allows adding additional charges. "Submit" creates the share account in "Submitted and Pending Approval" status. The Share Account Detail page displays the share account number, product name, client name, status badge (Submitted and Pending Approval, Approved, Active, Closed, Rejected), total approved shares, total pending shares, and unit price. Action buttons change based on status: for Pending — Approve (with Approved Shares and Approved Date), Reject; for Approved — Activate, Undo Approval; for Active — Apply Additional Shares (purchase more shares), Redeem Shares (sell back shares — the redemption amount is calculated as shares multiplied by current unit price and credited to the linked savings account), Close. Tabs show: Purchased Shares (table of all purchase and redemption transactions with Date, Type, Number of Shares, Unit Price, Amount, and Status), Dividends (posted dividend distributions with date, amount per share, and total amount), and Charges.

### 17. Fixed & Recurring Deposit Accounts

Fixed Deposit (FD) and Recurring Deposit (RD) accounts are created from the Client Detail page. The FD Account creation form includes Fixed Deposit Product (dropdown), Deposit Amount (numeric input), Deposit Period (numeric input and unit dropdown — Days, Months, Years), and maturity instructions (Transfer to Savings, Re-Invest, or Re-Invest Principal and Transfer Interest). The FD Account Detail page shows the deposit amount, maturity date, maturity amount, interest rate, and status with action buttons for Approve, Activate, Premature Close, and Close on Maturity. The RD Account creation form includes Recurring Deposit Product (dropdown), Mandatory Deposit Amount (numeric input per installment), Deposit Period, Deposit Frequency (dropdown — Daily, Weekly, Monthly, Quarterly, Yearly), and Expected First Deposit On date. The RD Account Detail page shows the deposit schedule, total deposits made, maturity details, interest rate, and status with action buttons for Approve, Activate, Deposit (to make recurring deposits), Premature Close, and Close on Maturity. Both FD and RD detail pages include tabs for Summary (account and interest details), Transactions (table of deposit and interest posting transactions), and Charges. Interest rates are determined by the Interest Rate Chart configured on the product, with tiered rates based on deposit amount and term length.

### 18. Accounting - Chart of Accounts

The Chart of Accounts page is accessed from the Accounting menu in the top navigation bar or the left sidebar. The breadcrumb shows Home / Accounting / Chart of Accounts. The page displays a hierarchical tree view or table of all General Ledger (GL) accounts organized by account type: Assets, Liabilities, Equity, Income, and Expenses. Each row shows the GL Code, Account Name, Account Type (with color coding — Assets in blue, Liabilities in red, Equity in green, Income in purple, Expenses in orange), Manual Entries Allowed indicator, Usage (Header or Detail), and Description. Header accounts serve as grouping parents while Detail accounts are used for actual journal postings. A "+ Create GL Account" button opens the Create GL Account form with fields for Account Type (dropdown — Asset, Liability, Equity, Income, Expense, required), Parent Account (dropdown listing header accounts of the same type), GL Code (text input, required, must be unique), Account Name (text input, required), Account Usage (radio buttons — Header, Detail), Manual Entries Allowed (checkbox), Description (textarea), and Tag (dropdown). Required fields show validation errors if empty. If a duplicate GL Code is entered, an error is displayed. "Submit" creates the GL account. Clicking an account name opens its detail view with Edit and Delete options.

### 19. Accounting - Journal Entries & Closures

The Journal Entries page is accessed from the Accounting menu in the top navigation bar. The breadcrumb shows Home / Accounting / Journal Entries. The page displays a data table of recent journal entries with columns for Entry ID, Office, Transaction Date, Created Date, Transaction ID, Type (Debit/Credit), GL Account, and Amount. A filter bar at the top allows filtering by Office (dropdown), GL Account (dropdown), Date Range (from/to date pickers), Transaction ID (text input), and Entry Type. A "+ Add Journal Entry" button opens the Create Journal Entry form with fields for Office (dropdown, required), Currency (dropdown, required), Reference Number (text input), Transaction Date (date picker, required), Payment Type (dropdown), and payment detail fields (Account Number, Check Number, Receipt Number, Routing Code, Bank Number). The entry lines section allows adding multiple debit and credit lines: each line has a GL Account (dropdown with search/autocomplete) and Amount (numeric input). An "Add Row" button adds additional lines. A running total display shows the debit total, credit total, and difference — the total debits must equal total credits for the entry to balance. A Comments field (textarea) allows adding a description. If totals don't balance, a validation error prevents submission. "Submit" posts the journal entry. The Closing Entries page (also under Accounting) displays a table of existing closures with columns for Office, Closing Date, Created Date, and Comments. A "+ Create Closure" button opens the Create Closure form with fields for Office (dropdown, required), Closing Date (date picker, required), and Comments (textarea). Creating a closure prevents journal entries from being posted for dates on or before the closing date, ensuring the integrity of completed accounting periods.

### 20. Accounting Rules & Financial Activity Mappings

The Accounting Rules page is accessed from the Accounting menu in the top navigation bar. The breadcrumb shows Home / Accounting / Accounting Rules. The page displays a data table of all configured accounting rules with columns for Rule Name (clickable link), Office, Debit Account(s), and Credit Account(s). A "+ Create Rule" button opens the Create Accounting Rule form with fields for Office (dropdown — specifies which office the rule applies to, or blank for all offices), Rule Name (text input, required), Debit Tags / Debit Account (dropdown or multi-select listing GL accounts to debit), Allow Multiple Debit Entries (checkbox), Credit Tags / Credit Account (dropdown or multi-select listing GL accounts to credit), and Allow Multiple Credit Entries (checkbox). Accounting rules automate journal entry creation for certain transaction types by pre-defining the debit and credit GL account mappings. "Submit" creates the rule. Clicking a rule opens its detail view with Edit and Delete options. The Financial Activity Mappings page (also under Accounting) displays a table mapping financial activities to GL accounts. Each row shows the Financial Activity (e.g., Asset Transfer, Liability Transfer, Cash at Mainvault, Cash at Teller, Opening Balances Transfer, Fund Source, Payable Dividends, Asset Fund Source, Overdraft Portfolio) and the mapped GL Account. A "+ Create Mapping" button opens the Create Mapping form with Financial Activity (dropdown listing all unmapped activities) and GL Account (dropdown). Each financial activity can only be mapped once. These mappings enable the system to automatically post accounting entries for internal transfers, teller operations, and other system-level financial activities.

### 21. Provisioning

The Provisioning pages are accessed from the Accounting menu in the top navigation bar. The Provisioning Criteria page displays a data table of all defined provisioning criteria with columns for Criteria Name (clickable link) and Created Date. A "+ Create" button opens the Create Provisioning Criteria form with Criteria Name (text input, required) and a definitions table where each row specifies: Loan Product (dropdown), Category (dropdown — e.g., STANDARD, SUB-STANDARD, DOUBTFUL, LOSS based on delinquency classification), Minimum Age (numeric input — minimum overdue days), Maximum Age (numeric input — maximum overdue days), Provisioning Percentage (numeric input — the percentage of outstanding principal to provision), Liability Account (GL account dropdown for the provisioning liability), and Expense Account (GL account dropdown for the provisioning expense). Multiple rows can be added to cover different overdue ranges and product combinations. "Submit" creates the criteria. The Provisioning Entries page lists all generated provisioning journal entries with columns for Entry Date, Journal Entry Created, Created By, and actions. A "+ Create Provisioning Entry" button generates new provisioning entries based on the current loan portfolio status and configured criteria — the system calculates the required provision amount for each loan category and creates the corresponding journal entries. Each entry can be reviewed (showing the detailed breakdown by loan product and category) or recreated.

### 22. Offices

The Offices page is accessed from the Admin menu in the top navigation bar under Organization. The breadcrumb shows Home / Organization / Offices. The page displays a hierarchical tree view or table of all offices showing Office Name (clickable link), External ID, Parent Office, Opening Date, and hierarchy. A "+ Create Office" button opens the Create Office form with fields for Office Name (text input, required), Parent Office (dropdown listing existing offices for hierarchy, required — Head Office is the root), Opened On Date (date picker, required), and External ID (text input, optional). Required fields show validation errors if empty. Upon submission, the new office is created under the selected parent in the organizational hierarchy. The Office Detail page shows the office information and provides Edit option. Offices define the branch structure — clients, groups, staff, and accounting operations are all associated with specific offices.

### 23. Employees

The Employees page is accessed from the Admin menu in the top navigation bar under Organization. The breadcrumb shows Home / Organization / Employees. The page displays a data table of all staff members with columns for Name (clickable link), Office, Is Loan Officer indicator, and Status (Active, Inactive). A "+ Create Employee" button opens the Create Employee form with fields for Office (dropdown, required), First Name (text input, required), Last Name (text input, required), Is Loan Officer (checkbox), Mobile Number (text input), Is Active (checkbox, defaults to true), Joining Date (date picker), and External ID (text input). Required fields show validation errors if empty. "Submit" creates the staff record. Staff members marked as "Is Loan Officer" appear in the Loan Officer dropdown when creating loans and can be assigned to clients, groups, and centers for portfolio management. The Staff Detail page shows the employee's information with an Edit option.

### 24. Teller & Cashier Management

The Teller / Cashier Management page is accessed from the Admin menu in the top navigation bar under Organization. The breadcrumb shows Home / Organization / Tellers. The Tellers page displays a data table of all configured tellers with columns for Teller Name (clickable link), Office, Status (Active, Inactive), and Description. A "+ Create Teller" button opens the Create Teller form with fields for Office (dropdown, required), Teller Name (text input, required), Description (textarea), Start Date (date picker, required), End Date (date picker, optional), and Status (dropdown — Active, Inactive). "Submit" creates the teller. The Teller Detail page displays teller information and provides an Edit option. Below the teller details, a Cashiers section lists all cashier assignments for that teller with columns for Cashier Name, Start Date, End Date, Is Full Day, and Status. A "+ Allocate Cashier" button opens the Allocate Cashier form with fields for Staff (dropdown listing employees, required), Start Date (date picker, required), End Date (date picker), Is Full Day (checkbox), and Description. Cashier allocation links a staff member to a teller terminal for a specified period. The Cashier Detail page shows the allocated cash — including Opening Balance, Cash In Hand, and transactions. Actions include Allocate Cash (add cash to the teller from the vault), Settle Cash (return cash from the teller to the vault — with Amount, Currency, and Transaction Date fields), and view the Cashier Transactions list showing all allocations, settlements, deposits, and withdrawals processed through that teller with Date, Type, Amount, and Running Balance columns.

### 25. Users & Roles

The Users page is accessed from the Admin menu in the top navigation bar. The breadcrumb shows Home / Admin / Users. The page displays a data table of all system users with columns for Username (clickable link), First Name, Last Name, Email, Office, and Status. A "+ Create User" button opens the Create User form with fields for Username (text input, required, must be unique), First Name (text input, required), Last Name (text input, required), Email (text input, required, validated for email format), Office (dropdown, required), Staff (dropdown to link user to a staff record), Password (password input, required, must meet configured password policy), Repeat Password (must match), Roles (multi-select checkboxes listing all available roles), Override Password Expiry Policy (checkbox), and Send Password to Email (checkbox). If the username already exists, an error is displayed. If passwords don't match, a validation error is shown. "Submit" creates the user. The Roles page (also under Admin) displays all roles with Name and Description columns. A "+ Create Role" button opens a form with Role Name and Description, then after creation the permissions page displays a categorized matrix of all system permissions (User Management, Portfolio, Organization, Accounting, Reports, etc.) with checkboxes for each permission. Enabling or disabling specific permissions controls what menu items, pages, and actions the role can access.

### 26. Reports

The Reports section is accessed from the Reports menu in the top navigation bar. The breadcrumb shows Home / Reports. The page lists available reports grouped by category: All, Clients, Loans, Savings, Funds, Accounting, and XBRL. Each report row shows Name (clickable link), Type (Table, Pentaho, Chart), and Category (Loan, Savings, Fund, Accounting, Client). Clicking a report opens a parameters form tailored to that report type, with fields such as Office (dropdown), Branch (dropdown), Currency (dropdown), Loan Product (dropdown), Date Range (From Date and To Date pickers), Loan Officer (dropdown), Fund (dropdown), and other report-specific parameters. Required parameter fields show validation errors if empty. A "Run Report" button generates the report which displays as a data table in the main content area with sorting capabilities and pagination. Report output options include viewing in a table format on screen, exporting to Excel (XLS/XLSX), exporting to CSV, and exporting to PDF. Common reports include Loans Awaiting Disbursal, Loans Awaiting Disbursal Summary, Loans Pending Approval, Active Loans — Summary, Active Loans — Details, Obligation Met Loans Details, Portfolio at Risk, and Portfolio at Risk by Branch.

### 27. Account Transfers & Standing Instructions

The Account Transfers function is accessible from the Savings Account Detail page or from the Self Service menu in the top navigation bar. The transfer form contains fields for From Office (dropdown), From Client (dropdown), From Account Type (dropdown — Savings Account, Loan Account), From Account (dropdown listing the client's accounts), To Office (dropdown), To Client (dropdown), To Account Type (dropdown — Savings Account, Loan Account), To Account (dropdown listing target accounts — can be same client or different clients), Transfer Amount (numeric input, required), Transfer Date (date picker, required), and Description (textarea). If the transfer amount exceeds the available balance, an error is shown. "Submit" processes the transfer, debiting the source and crediting the destination. The Standing Instructions page displays a data table of all configured recurring transfers with columns for Name, From Client, From Account, To Client, To Account, Amount, Validity (From/Till dates), and Status (Active, Disabled). A "+ Create Standing Instruction" button opens the Create Standing Instruction form with fields for Name (text input, required), From Office, From Client, From Account Type, From Account, To Office, To Client, To Account Type, To Account, Transfer Type, Priority (dropdown), Instruction Type (dropdown — Fixed, Dues), Amount (numeric input), Validity From Date (date picker), Validity Till Date (date picker), Recurrence Type (dropdown — Periodic, As Per Dues), Recurrence Frequency (dropdown), and Recurrence Interval (numeric). Standing instructions can be enabled, disabled, or deleted from the listing.

### 28. Tax Management

The Tax Management pages are accessed from the left sidebar under Products (Manage Tax Configurations). The breadcrumb shows Home / Products / Manage Tax Configurations. The Tax Components page displays a data table of all tax components with columns for Name (clickable link), Percentage, Debit Account Type, Debit Account, Credit Account Type, Credit Account, and Start Date. A "+ Create Tax Component" button opens the Create Tax Component form with fields for Name (text input, required), Percentage (numeric input, required — the tax rate to apply), Debit Account Type (dropdown — Asset, Liability, Equity, Income, Expense) and Debit Account (GL account dropdown), Credit Account Type (dropdown) and Credit Account (GL account dropdown), and Start Date (date picker, required). "Submit" creates the tax component. The Tax Groups page displays a data table of tax groups with columns for Name (clickable link) and associated components. A "+ Create Tax Group" button opens the Create Tax Group form with Name (text input, required) and a Tax Components section allowing multiple tax components to be added with Start Date and End Date for each. Tax groups combine multiple tax components into a single applicable rate. Tax groups are referenced by Savings Products (Enable Withhold Tax checkbox with Tax Group dropdown) and Charge definitions — when a product has tax enabled, the system automatically applies the configured tax rate on interest income or fees and posts the corresponding journal entries to the mapped GL accounts.

### 29. Organization Settings

The Organization section is accessed from the Admin menu in the top navigation bar. The breadcrumb shows Home / Organization. It provides access to several configuration pages. The Holidays page displays a table of configured holidays with columns for Name, Start Date, To Date, Status (Active, Pending), and Rescheduling Type. A "+ Create Holiday" button opens the Create Holiday form with fields for Name (text input, required), From Date (date picker, required), To Date (date picker, required), Repayments Rescheduled To (date picker), Rescheduling Type (dropdown), Description (textarea), and applicable Offices (multi-select). Holidays affect loan repayment schedules — installments falling on holidays are automatically rescheduled per the configured type. The Working Days page allows configuring which days of the week are working days using checkboxes for Monday through Sunday and a Repayment Rescheduling dropdown for handling repayments on non-working days. The Currencies page allows selecting active currencies from available currencies using an add/remove interface, each showing Code, Name, and Decimal Places. The Funds page manages funding sources for loans with a table listing Fund Name and External ID, and a Create Fund form with Fund Name (required) and External ID fields. The Payment Types page lists all configured payment methods (Cash, Bank Transfer, Mobile Money, Check, Receipt) with Name, Description, Is Cash Payment indicator, and Position columns; a "+ Create" button creates new payment types which appear as dropdown options in all transaction forms across the system. The Bulk Import page (breadcrumb: Home / Organization / Bulk Import) allows importing data for multiple entity types — Clients, Groups, Centers, Offices, Staff, Users, Loans, Savings, and more — each with a Download template and Upload interface.

### 30. System Administration

The Admin menu in the top navigation bar provides access to several system management pages. The Manage Scheduler Jobs page (breadcrumb: Home / Admin / Scheduler Jobs) displays a table of all batch jobs with columns for Job Name (Display Name), Is Active (toggle), CRON Expression, Previous Run Status (Success/Failed with job run start/end times), Next Run Time, Currently Running indicator, and action buttons. Jobs include Apply Annual Fee, Add Accrual Transactions, Apply Penalty to Overdue Loans, Update Loan Summary, Post Interest for Savings, Transfer Fees, Generate Loan Loss Provisioning, and others. Each job can be toggled active/inactive and its CRON expression edited. A global scheduler toggle (Start/Stop) controls whether scheduled jobs execute. The Global Configuration page displays a table of system-wide feature flags with columns for Configuration Name, Enabled (toggle), Value, and Description. Key settings include maker-checker (4-eyes approval), reschedule-future-repayments, allow-transactions-on-non-workingday, reschedule-repayments-on-holidays, savings-interest-posting-current-period-end, financial-year-beginning-month, and meetings-mandatory-for-jlg-loans. The Manage Codes page lists all code lists (Client Type, Gender, Address Type, Loan Purpose, etc.) with Name and Is System Defined columns; clicking a code opens its values for adding, editing, reordering, and deactivating entries that populate dropdown options throughout the system. The Manage Data Tables page lists custom data tables that extend core entities (valid application tables: m_client, m_group, m_loan, m_office, m_saving_account, m_product_loan, m_savings_product) with custom fields; the Create form includes Data Table Name, Application Table Name (dropdown), Multi Row (checkbox), and column definitions (Name, Type — string/number/boolean/decimal/date/datetime/text/dropdown, Length, Is Mandatory, Is Unique). The Audit Trails page displays all system actions with columns for Action Name, Entity Name, Resource ID, Maker, Made On Date, Checker, Checked On Date, Processing Result (Pending, Approved, Rejected), Office Name, and Client/Loan/Savings details; filters allow searching by Action Name, Entity Name, Resource ID, Maker ID, Checker ID, Office, and Date Range; when maker-checker is enabled, pending commands appear with Approve/Reject buttons.

### Logout

The logout function is accessible from the user profile icon in the top-right corner of the navigation bar. Clicking the user icon reveals a dropdown menu with options for Profile Settings and Log Out. Clicking Log Out terminates the authenticated session, clears the authentication token stored in the browser, and redirects the user to the login page. After logout, any attempt to navigate to an authenticated page redirects to the login page.
